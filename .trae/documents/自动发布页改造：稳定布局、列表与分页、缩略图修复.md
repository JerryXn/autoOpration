## 问题诊断
- Tab 切换时页面上下跳动：两个 Pane 高度不一致，容器随内容高度变化导致布局抖动。
- 自动发布以卡片“块状”展示，不便于分页和信息密集展示。
- 列表中图片缩略图丢失：本地文件路径转为 `file://` URL 的方式存在兼容问题；仅返回封面，不含多图。
- 顶部标题“自动发布”与 Tab 文案重复，需要去掉标题，仅保留 Tab。

## 改造目标
- 固定自动发布区域高度，Tab 切换不影响整体页面高度。
- 自动发布页改为“表格列表 + 分页控件”，每行展示：缩略图（封面）、标题、主题、平台、状态、配图缩略图（多张）、创建时间。
- 支持分页（page、pageSize），展示总数并可翻页。
- 修复缩略图为可见状态，文件来源为本地 `file://`，同时包含多张配图缩略图。
- 去掉顶部 H1 的“自动发布”，仅保留 Tab 的“自动发布”。

## 后端接口调整
- 新增/改造 `list-parsed-contents(platformCode, page, pageSize)`：
  - 入参：`platformCode`（可选，用于筛选当前平台）、`page`（默认 1）、`pageSize`（默认 20）。
  - 返回：`{ total, items: [{ id, platform_code, type, title, subject, status, created_at, coverThumbUrl, imageThumbUrls: [] }] }`。
  - 数据获取：
    - 查询 `parsed_contents` 按 `user_id`（必选）、`platform_code`（可选）筛选，`ORDER BY id DESC`，`LIMIT ? OFFSET ?`。
    - 解析 `cover_refs` 与 `image_refs`（JSON 数组），对每个 id 查询 `media_files.storage_path`，生成 `file://` 绝对 URL 列表。
  - 文件 URL 生成：使用 `url.pathToFileURL(abs).href`，避免手工拼接导致不可见问题。
  - 保持现有权限与索引：使用 `idx_pc_user`、`idx_pc_platform`，分页查询高效。

## 前端 UI 与交互
- Tab 区域与 Pane 高度稳定：
  - 为 `#autopub-card` 设置固定 `min-height`（如 520px）或固定高度，内部 `import-pane`、`publish-pane` 使用相同高度并 `overflow:auto`，避免跳动。
  - Tab 切换时仅切换 `display` 或 `visibility`，不改变容器高度。
- 自动发布 Tab 列表：
  - 结构：表格 `<table>` + `<thead>/<tbody>`；底部添加分页控件（上一页/下一页/页码显示）。
  - 列：封面缩略图、标题、主题、平台、状态、配图缩略图组（最多前 4-6 张）、创建时间。
  - 缩略图大小：封面统一 60×80；配图缩略图 40×40 横向排列（超出显示“+N”）。
  - 列表加载：进入“自动发布” Tab 或分页切换时调用 `listParsedContents(state.selectedPlatform, page, pageSize)` 并渲染。
- 顶部标题调整：
  - `ops.js` 不再将 `opsTitle` 改为“自动发布”；保持原“功能选择”，或置空标题；视觉由 Tab 文案体现。
