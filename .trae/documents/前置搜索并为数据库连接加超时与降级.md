## 问题
- 入口在 `paginate → db connect` 后被中断，导致看不到 `search_notes` 调用与返回；数据库连接可能阻塞（本地 MySQL 未就绪），搜索调用位于连接之后。

## 方案
### 1. 调用顺序调整
- 在 `cookie_built → search_keyword → comments_config` 后，立即执行：
  - `api.search_notes_with_body(...)`（8s 超时）
  - 打印 `search_notes begin/resp`（已增加调用与返回打印）
- 解析结果并进入详情页执行动作。
- 仅当 `--record` 为真时，随后连接数据库并写入；不记录时跳过连接，避免阻塞。

### 2. 数据库连接健壮性
- 为 `MySQLClient.connect()` 增加超时（如 3s），通过 `connect_timeout` 传入驱动；失败时：
  - 打印 `db connect error` 简要信息
  - 自动降级为不记录（继续动作流程），或改用轻量库 `auto_test`（仅越过检查，不写入）。

### 3. 页面回退强化
- 当 curl 缺失或返回空：
  - 仍然进入 `/explore`，滚动与等待 `/feed` 响应
  - 采集 `a[href*="/explore/"]` 提取 `note_id` 列表
  - 打印 `search_notes_captured ok {count}` 并进入详情执行动作

### 4. 日志增强
- 入口阶段：`api search_notes begin/resp`（已加）
- 评论分页：`paginate_comments begin/resp`（已加）
- 连接失败：`db connect error` 简洁返回

### 5. 验证
- 运行：`MYSQL_DATABASE=auto_test XHS_HEADLESS=false python -m auto_opration_python.src.main_xhs --like --keyword=1`
- 预期：出现搜索调用与返回，随后进入详情并执行点赞；数据库未就绪时也不会阻塞搜索与动作。
