## 目标
- 取消 MCP 地址输入框。
- 将类型、关键词、浏览、记录浏览信息、排序、发布时间、搜索范围、位置距离集中到“搜索区域”。
- 新增“风控区域”，包含随机延迟(ms)与频率/数量限制。
- 将 lv1/lv2/lv3 权限区块整体下移到“风控区域”之后。
- 压缩样式与排版，使页面在默认窗口内尽量无需滚动。

## 页面结构调整（HTML）
- 在 `src/index.html` 的自动运营卡片内：
  - 删除 MCP 地址输入：移除 `#autoop-mcp-url` 所在的 `<div>`。
  - 新增 `<section id="search-area" class="area">`，内部按两列/三列网格排版：
    - `类型(autoop-type)`、`关键词(autoop-keywords)`、`浏览(autoop-browse)`、`记录浏览信息(autoop-record)`、`排序(autoop-sort)`、`发布时间(autoop-publish-time)`、`搜索范围(autoop-scope)`、`位置距离(autoop-distance)`。
  - 新增 `<section id="risk-area" class="area">`：
    - `随机延迟(ms)(autoop-delay-range)`、`每次数量(autoop-limit-count)`、`每分钟上限(autoop-limit-permin)`、`每小时上限(autoop-limit-perhour)`。
  - 将现有 `lv1-area / lv2-area / lv3-area` 区块整体置于 `risk-area` 之后。

## 脚本调整（JS）
- 在 `src/views/autoop.js`：
  - 移除对 `autoop-mcp-url` 的获取与引用（变量与任何读取）。
  - 保持事件绑定和 `startAutoOp(payload)` 的现有字段（`limits`、`filters`、`actions`）不变。
  - 保留并兼容现有权限提示与悬停提示逻辑。

## 样式压缩（CSS）
- 在 `src/styles/app.css`：
  - 为 `.area`、label 与网格整体减小 padding/margin（如 `.area { padding:10px; }`）。
  - 调整搜索与风控区域为两列/三列栅格，减少竖向空隙。
  - 去掉 `#autoop-card` 的固定最大高度与滚动（`overflow-y:auto`），改为自适应，配合压缩后的布局尽量单屏展示。
  - 若仍超屏：进一步缩小字号与间距，或将某些字段并排展示（inline label + input）。

## 验证
- 启动应用，进入自动运营页面：
  - 确认 MCP 地址不再显示；
  - “搜索区域”包含所列字段，布局紧凑；
  - “风控区域”包含延迟与频率限制；
  - `lv1/lv2/lv3` 区块位于风控区域后；
  - 默认窗口下页面无需滚动或仅最小滚动。
- 功能检查：点击“执行”时 payload 未缺失必要字段（尤其 `limits.delayRange`、`filters.*`）。

## 交付项
- 更新后的 `src/index.html` 与 `src/styles/app.css` 布局与样式。
- 更新后的 `src/views/autoop.js` 删除 MCP 地址引用并保持功能兼容。
- 说明变更点，便于后续继续微调排版与文案。
